Problem Description

Introduction

In many multiagent systems, a fruitful interaction for each single agent or for the whole is made more difficult by the fact that the environment is unpredictable and the information available about the world and other agents is noisy and imperfect.

In order to model this situation, a Decentralized Partially Observable MDP (Dec-POMDP) model is introduced, as in [OA-2015] it is a tuple (D, S, A, p, r, O, O,γ,T,b0)(\textit{D, S, \textbf{A}, p, r, O, \textbf{O}}, \gamma, T, b_0)(D, S, A, p, r, O, O,γ,T,b0​) where

  • A\textit{\textbf{A}} A is the set of joint actions the agents can perform over which a joint policy π(a∣o)\pi(a|o)π(a∣o) is defined

A key assumption of Dec-POMDPs is that the agents can neither see the actual state of the system nor explicitly communicate their noisy observations with each other due to communication cost, latency or noise. The Dec-POMDP framework can model communication implicitly through the actions, states and observations. This means that communication has no special semantics in it, a visualization of these concepts is presented in [Fig. 1].

The decentralized framework is useful in many real scenarios since in a centralized counterpart, the state and action spaces would grow exponentially with the number of agents and be totally intractable.

Due to the Partial Observability property, the agents can maintain an internal representation and description of histories, called internal states. When the transition dynamics is available, it is possible to maintain a belief over states, with one statistic for each agent. That is, they can use the history of observations to estimate the probability of each state and use this information to decide upon an action:

∀stb(st)≜p(st∣H0:t),H0:t=(ot,at−1,…,a0)\forall s_t \quad b(s_t ) \triangleq p(s_t | H_{0:t}), \quad H_{0:t} = (o_t, a_{t-1}, \dots, a_0)∀st​b(st​)≜p(st​∣H0:t​),H0:t​=(ot​,at−1​,…,a0​)

where the belief space is a time-invariant space, differently from other statistics.

The only sufficient statistic for Dec-POMDPs known so far relies on the occupancy state , which in the individual case is given by σti(s,o−i)=p(s,o−i∣oi,σt)\sigma_t^i(s, o^{-i}) = p(s, o^{-i}|o^i, \sigma_t)σti​(s,o−i)=p(s,o−i∣oi,σt​).

Differently from control theory, usually in POMDPs there is no certainty equivalence approach: that is, the agents explicitly reason about believes and their uncertainty and do not behave as they were a certain quantity. As a result, agents can reason about information gathering when beneficial and select actions that will provide information about the state.

Factored and Networked Models

Some of the spaces in the model might be factored into different components. In this case we talk about factored Dec-POMDPs.

In the case of agent-wise factored models, the world state can be factored into different components, one for each agent and one more for the environment itself, S=S0×S1×⋯×S∣D∣S =S^0 \times S^1 \times \dots \times S^{|D|}S=S0×S1×⋯×S∣D∣. Each agent can fully observe some subsets of these state factors according to some indices included in the set FiF_iFi​.

As such, the local states for the agent iii are Si=×jSj  ∀j∈FiS_i = \times_j S^j \; \forall j \in F_iSi​=×j​Sj∀j∈Fi​. An example of factorization could include factors for the location of each agent as well as the location of several movable objects. All the factors about partial or location-dependent information about one agent or others might be factorized as well.

A factored Dec-POMDP is said to be transition independent (TI) if p(s′∣s,a)=∏i∣D∣pi(si′∣si,ai)p(s'|s,\bf{a} ) = \prod_i^{|D|}p_i(s'_i|s_i, a_i)p(s′∣s,a)=∏i∣D∣​pi​(si′​∣si​,ai​), the same goes for observations with observation independent (OI) systems. In such cases the agents never affect each other or each other’s observations.

By exploiting the constraint structure induced by the factorization, it is useful to define Coordination (Hyper-)Graphs (CGs), whose definition is based on the fact that even thought al agents are relevant for the total payoff, not all agents directly interact with each other, leading to a form of conditional independence. A CG is defined as a tuple CG=(D,A,{u1,…,uρ})CG=(D,A, \{u^1, \dots, u^\rho\})CG=(D,A,{u1,…,uρ}) where

  • D,AD, AD,A are the sets of agents and joint actions
  • {u1,…,uρ}\{u^1, \dots, u^\rho\}{u1,…,uρ} is the set of local payoff functions

Each local payoff ueu^eue maps actions of a subset of agents e⊆De \subseteq De⊆D to payoffs, called the scope of the function ue(ae)u^e(a_e)ue(ae​). The goal in a CG is to find the joint action with the highest total payoff u(a)=∑e∈Eue(ae)u(a) = \sum_{e \in E} u^e(a_e)u(a)=∑e∈E​ue(ae​). An example of this structure is presented in [Fig. 1].

Here, the local reward functions should not be confused with individual reward functions in a system with self-interested agents.

The framework of CG can be applied to Dec-POMDPs by replacing actions with policies and forming value-factored Dec-POMDPs, thus stating that the value function can be additively decomposed into local values

Now, letting NiN_iNi​ being the neighbours of agent iii in the interaction graph, including itself, it is possible to introduce a local neighbourhood utility as the expected return of NiN_iNi​, V(π,Ni)=∑e:i∈eVe(πe)V(\pi, N_i) = \sum_{e : i \in e} V^e(\pi_e)V(π,Ni​)=∑e:i∈e​Ve(πe​). The LoI property is pretty restrictive, and a broader class of problems can be introduced while it is possible to see to what extent the property still holds.

Continuous-time POMDPs (CT-POMDPs)

A common assumption in solving Dec-POMDPs is that planning takes place in a centralized (offline) manner even though agents execute actions in a decentralized fashion (online), as in [DABC-2016]. In this paradigm, a centralized algorithm maintains the total available information and performs policy search in the space of separable joint policies. Thus, the separable joint decision rule choices are based only on the exhaustive information available to the centralized algorithm or on statistics derived from that information. The statistics summarizing the exhaustive information available to the centralized algorithm are called information states (IS). Complete information states represent the trivial case of exhaustive data.

A complete information state (CIS) is defined as itC=(b0,π0:t−1)i_t^C = (b_0, \pi_{0:t-1})itC​=(b0​,π0:t−1​) and is a length-ttt sequence of separable joint decision rules starting from the initial belief state b0b_0b0​.

The collection of random variables {it}\{i_t\}{it​} taking values in the state space SSS defines an information-state MDP (i-MDP). In such MDP, states are information states and actions are separable joint decision rules. The i-MDP is deterministic since it+1=p(it,δt)i_{t+1} = p(i_t, \delta_t)it+1​=p(it​,δt​) and these state space is implicit: The space itself is too large but the information states are generated as they are explored.

The use of CIS has poor behaviour since they usually result in redundant and useless computations any-time some information about iCi^CiC has to be used, such in R(iC,δ)R(i^C, \delta)R(iC,δ)

Dynamic Bayesian Networks (DBNs)

A Bayesian network is a representational framework for compactly representing a PD in factored form, and as in [OSWV-2008], this formalism can be useful in fDec-POMDPs, since he transition and observation model can be compactly represented by exploiting conditional independence between variables.

Formally, a Bayes network is a directed acyclic graph in which vertices correspond to random variables and an edge between two variables is a direct probabilistic dependency between them. In Dynamic Bayesian Networks (DBNs), vertices represent the system’s state at different time points and arcs represent dependencies across time points. In these temporal networks, each vertex’s parent is its temporal predecessor, the conditional distributions are transition PD and the marginal distributions are distributions over initial states. Each node with incoming edges has a conditional probability table (CPT) associated with it. Even if the size of such table is exponential in the number of parents, this number is typically a subset of all state factors and actions, leading to a model exponentially smaller than the flat one. The rewards are included in the model via reward nodes with associated conditional reward tables (CRT). An example is shown in [Fig. 1] for a fire fighting problem.

Cooperative Bayesian Games (CBGs)

A Bayesian Game (BG) is an extension of a strategic form game in which agents have private information. A collaborative BG (CBG) is a BG with identical payoffs.

In Dec-POMDPs, give the initial state distribution μ\muμ and past joint policy ϕt\phi_tϕt​, the choice of finding an optimal policy can be modelled as a collaborative CBG.

Problem Complexity

According to [OA-2015] the problem of finding the optimal solution for a finite-horizon Dec-POMDP with ∣D∣>2|D| > 2∣D∣>2 is NEXP-complete (non-deterministic exponential time), while the infinite-horizon is undecidable.

According to [RGS-2003] even finding an ϵ\epsilonϵ-approximate solution is NEXP-complete. That is even finding a joint policy whose value is V(π)≥V(π∗)−ϵV(\pi) \geq V(\pi^*) - \epsilonV(π)≥V(π∗)−ϵ is also intractable.

Very strong assumptions need to be imposed in order to lower the complexity (in the finite-horizon case)

Communication

It is possible to incorporate communication between agents. As said before, the agents might communicate by means of their regular actions and observations, this is referred to as implicit communication protocols, with this additional actions and observations being added to the initial sets. On the other hand, when the framework has a separate set for communication actions, the communication is said to be explicit.

There are several taxonomies for explicit communication (i.e. Tell, Query, Sync or One-way, Two-way, Acknowledged communication) but in any case it would be required to incorporate into the model a set of messages and a related cost function for the communication to happen. Moreover, a specific semantic should be added in order for the communication to be interpreted correctly, or the most effective semantic should be added to the optimization goals. Finally, some noise or delay in the communication should be added, otherwise the communication is assumed to be perfect.

In the case of instantaneous, noise free and cost free communication the framework is that of Multi-agent POMDPs (MPOMDPs), a joint communication policy that shares the local observations at each stage was proven to be optimal.
