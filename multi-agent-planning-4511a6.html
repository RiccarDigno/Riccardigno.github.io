Introduction

Once planning is taken into account, due to the partial observability of the environment, a belief over the value of the actions might be useful. An optimal joint policy for a certain belief state is the joint policy with the maximum value at that belief state.

As for planning, policies can be modelled as policy trees [Fig. 1], where the root is the action taken at the first time-step, the arcs represent observations. In finite-horizon cases, the depth of the tree corresponds to the horizon itself.

The value of a policy can be represented recursively as

Vπ(s,o)=R(s,π(o))+∑s′∈S∑o′∈OO(o′∣s′,π(o))p(s′∣s,π(o))V(s′,π(o′))V^{\pi}(s, o) = R(s, \pi(o)) + \sum_{s' \in S} \sum_{o' \in O} O(o'| s', \pi(o)) p(s'|s,\pi(o))V(s', \pi(o'))Vπ(s,o)=R(s,π(o))+s′∈S∑​o′∈O∑​O(o′∣s′,π(o))p(s′∣s,π(o))V(s′,π(o′))

The whole value is then given weighting these pairs according to the initial state distribution

V(π)=∑s0∈Sb0(s0)Vπ(s0,o0)V(\pi) = \sum_{s_0 \in S}b_0(s_0)V^{\pi}(s_0, o_0)V(π)=s0​∈S∑​b0​(s0​)Vπ(s0​,o0​)

As for planning, the main reference is [OA-2015], which takes into account decentralized deterministic policies in the finite horizon (FH) case and infinite horizon one (IH) (according to [OSV-2008] there exists at least one optimal deterministic solution for finite-horizon cases).

Solutions

Dynamic Programming (DP)

Exact DP [OA-2015]

DP for Dec-POMDPs needs to maintain partial policies, called sub-tree policies, and values for them.

Additionally, a multiagent belief is sufficient to predict the next multiagent belief via Bayesian updates.

The core idea of DP for Dec-POMDPs is to incrementally construct sets of longer sub-tree policies Qi1,…QiT∀i∈DQ^1_i, \dots Q^T_i \quad \forall i \in DQi1​,…QiT​∀i∈D by using backups, that construct Qiτ+1Q_i^{\tau +1} Qiτ+1​ from QiτQ^\tau_iQiτ​, and once the last backup step is completed, the optimal policy can be found by evaluating all induced joint policies π∈×iQiT\pi \in \times_i Q^T_iπ∈×i​QiT​ for the initial belief.

The exhaustive backup has exponential complexity. To counter this source of intractability, it is possible to prune dominated sub-tree policies, where qiτq_i^\tauqiτ​ is dominated if it is not maximizing at any point in the multiagent belief space. It is then necessary to store all the values V(st,qτ)V(s_t, q^\tau)V(st​,qτ). Removal of a dominated sub-tree policy may cause another agents to become dominated. Therefore DP iterates over agents until no further pruning is possible with iterated elimination of dominated policies. This leads to

PBDP [SC-2006]

The main problem of exact DP is that the set of maintained sub-tree policies grow very quickly. In fact, DP only removes qiτq_i^\tauqiτ​ that are not maximizing at any point in the multi-agent belief state. As in [SC-2006], the Point-based DP (PBDP) improve pruning by considering only a subset of reachable belief points Bi⊂Δ(S×Q−iτ)B_i \subset \Delta(S \times Q^\tau_{-i})Bi​⊂Δ(S×Q−iτ​) and only those sub-tree policies that maximise over that subset are kept.

In reachable belief points are defined as

where Γi,t:Oˉj,t→Qjτ\Gamma_{i,t}:\bar{O}_{j,t} \rightarrow Q_j^\tauΓi,t​:Oˉj,t​→Qjτ​ is a mapping from observation histories to sub-tree policies and Γ−i,t={Γj,t}j≠i\Gamma_{-i, t} = \{\Gamma_{j,t}\}_{j \neq i}Γ−i,t​={Γj,t​}j​=i​. A belief is reachable if the two terms exist.

BDP [ACZ-2007]

Bounded DP (BDP) is a version of DP that can compute a bounded approximation and the primary idea presented in [ACZ-2007] is to prune out un-valuable branches, while guaranteeing a bound on the value of the error. Instead of pruning only dominated policies, BPD performs a more aggressive ϵ\epsilonϵ-pruning. In fact, the value of an agent’s belief is

V(bi)=max⁡qiτ∑s,q−iτbi(s,q−iτ)V(s,q−iτ,qiτ)V(b_i) = \max_{q_i^\tau}\sum_{s, q^\tau_{-i}} b_i(s,q_{-i}^\tau)V(s,q^\tau_{-i}, q^\tau_{i})V(bi​)=qiτ​max​s,q−iτ​∑​bi​(s,q−iτ​)V(s,q−iτ​,qiτ​)

It represents the maximum the agent can achieve given its set of policies weighted by the probability that the true state is ss s and the other agents act according to π−i\pi_{-i}π−i​. As b(si,π−i)b(s_i, \pi_{-i})b(si​,π−i​) depends on the policies of the other agents, all policies that may contribute to this term must be retained, the others may be removed or pruned. That is, the actions that have lower value for all possible states and actions of the other agents are removed. Instead of testing this inequality itself, the pruning goes for a stricter ϵ\epsilonϵ bound.

A main issue of this approach is that while all t+1t+1t+1 policies will be generated exhaustively, a majority of these will be later pruned.

IPG [ADZ-2009]

As a result in [ADZ-2009], the efficiency of dynamic programming is extended via a reachability analysis of the state space in the Incremental Policy Generation (IPG) algorithm. In fact, the action taken and the observation seen may limit the possible next states no matter what actions the other agents choose, allowing only policies that are useful for these possible states to be retained. This extension can be applied to all the known planning algorithms for for finite and infinite horizon.

MBDP [SZ-2007a]

Another way to approach this issue is devised in [SZ-2007a] and it is called Memory-Bounded DP (MBDP), which performs two approximations:

  • for each agent a fixed-size sub-tree policies QikQ^k_iQik​ is used according to a Max-Trees parameter
  • it does not sample reachable believes (which involves sampling past joint policies) but joint believes at which candidate sub-tree policies are evaluated thanks to heuristic joint policies.

In detail, MBDP starts with performing an exhaustive backup on the sets {Qi2}\{ Q^2_i\}{Qi2​} and in order to reduce the size to Max-Trees it uses heuristic policies to sample both joint-action observation histories and associated joint believes. At each joint belief it selects the joint sub-tree policy such that

q2=arg max⁡q2∈Q2V(bT−2,q2)q^2 = \argmax_{q^2 \in Q^2} V(b_{T-2}, q^2)q2=q2∈Q2argmax​V(bT−2​,q2)

This means that MBDP pretends that the resulting joint believes are revealed to the agents and it retains only the trees that have highest value at there joint believes.

This joint sub-tree policy is called point-based backup operation and it corresponds to solving a one-shot constraint optimization problem, or collaborative Bayesian Game, for each sub action.

PBPG [WZC-2010]

One problem with MBDP is that it produces a bounded pool of policies that are optimized with respect to a set of reachable belief states. Yet, the best policy for each belief state can be constructed directly rather then producing first a large set of candidates, as in [WZC-2010] with Point-based Policy Generation (PBPG).

Heuristic Methods (H)

MAA* [RN-2009]

One example of heuristic search is Multi-Agent A* (MAA*).

A common way to represent the forward temporal structure of a policy is to split it into decision rules δi\delta_iδi​ that specify the policy for each stage. An individual policy is then represented as a sequence of decision rules and a past joint policy at stage ttt is ϕt=(δ0,…,δt−1)\phi_t = (\delta_0, \dots, \delta_{t-1})ϕt​=(δ0​,…,δt−1​). Using this notation it is possible to grow a past joint policy by concatenation.

MAA* performs standard A* search as in [RN-2009]: it maintains a list of partial joint policies ϕt\phi_tϕt​ and their heuristic values V^(ϕt)\hat{V}(\phi_t)V^(ϕt​). On every iteration the node with highest value is selected and expanded, that is generating all child nodes and evaluating them. When using an admissible heuristic -a guaranteed overestimation- the heuristic values of the newly expanded policies are an upper bound to the true values and any lower bound that has been found can be used to prune the list.

In MAA*, V^(ϕt)=V0…t−1(ϕt)+V^t…T−1(ϕt)\hat{V}(\phi_t) = V^{0 \dots t-1}(\phi_t) + \hat{V}^{t \dots T-1}(\phi_t)V^(ϕt​)=V0…t−1(ϕt​)+V^t…T−1(ϕt​) where

  • V0…t−1(ϕt)V^{0 \dots t-1}(\phi_t)V0…t−1(ϕt​) is the actual expected reward
  • V^t…T−1(ϕt)\hat{V}^{t \dots T-1}(\phi_t)V^t…T−1(ϕt​) is a heuristic value for the remaining stages T−tT-tT−t

In order to mitigate the problem of the δt\delta_tδt​ growing doubly exponentially with ttt, several approximate methods have been introduced: lossless clustering of the histories [OWS-2009], incrementally expanding nodes only when needed [SOA-2011].

GMAA* [OSV-2008]

MAA* can be formulated to incorporate naturally FSCP in Generalized MAA* (GMAA*) [OSV-2008].

JESP [NTYPM-2003]

One other possible examples of heuristic approaches is Joint Equilibrium based Search for Policies (JESP) [NTYPM-2003] which relies on a process called alternating maximization. It is a procedure that computes a policy πi\pi_iπi​ by maximizing the joint reward while keeping the policies of the other agents fixed. Next, another agent is chosen to maximise the reward by finding its best response. This process is repeated until the joint policy converges to a Nash equilibrium, which is a local optimum, but this local optimum can be of course arbitrarily bad.

Moreover, rather than exhaustively loop over the policies available, it is possible to iteratively reformulate the problem as an augmented POMDP where the nominal state in augmented with the other observation histories, since these are fixed in each step.

FSCs Approaches in IH

Finite State Controllers (FSCs)

In the infinite-horizon case a tree-based representation of a policy requires the agent to perfectly remember the entire history in order to determine its next action. As an alternative, it is possible to consider a finite sets of internal states IiI_iIi​ which represent an agent finite memory. A natural representation for the policy in this form is finite-state controllers (FSCs). In FCSs there is an initial node and following the action selection at that node the controller transitions to the next node depending on the observation seen. For Dec-POMDPs, a set of controllers, one per agent, provides the joint policy.

💡

Note: The introduction of FSCs alters the multiagent decision problem, since the agents action were assumed to be based on histories, yet this is not the case for FSCs.

There exist two types of FCSs controllers as in [Fig. 2]:

  • Moore controllers: they associate actions with nodes
  • Mealy controllers: they associate actions with transitions

Deterministic controllers limit the value of policy represented as controllers, while stochastic controllers are able to produce higher-quality solutions with the same number of nodes. Stochastic controllers are similar to deterministic ones except for the probability distributions to transition and outputs.

💡

Note: It is always possible to convert a stochastic controller to a (larger) deterministic one.

Finally, it is important to notice that these solutions are able to represent just factored policies. Higher quality solutions could be achieved if correlation was permitted between agents.

In order to achieve this, controllers have been developed in order to make use of shared sources of randomness, called correlation devices, which allow a set of independent controllers to be correlated without sharing additional information.

One option is to introduce periodic FSCs as in [PP-2011], since they allow the use of much larger controller spaces with small complexity, efficient initialization and new algorithms.

A periodic FSC is composed of M layers of controller nodes, fully connected, with the last layer connected to the first. A single-layer periodic FSC is a standard FSC.

The approach of Policy Iteration (PI) [OA-2015] is similar to DP but with a different policy representation. In addition, there are two main differences with the finite case:

  • PI maintains a single controller for each agent and not different representations, moreover it considers the value of beginning execution from any node of the controller. That is, starting at each node can be interpreted as an infinite-horizon policy. Pruning can then take place over nodes of these controllers to remove dominated policies.
  • Rather than initializing the iteration process with the 1-step-to-go policies, PI can start from any initial joint controller.

The basic procedure of PI is trying to improve the joint controller by improving the controller maintained for each agent. The improvement is done via an exhaustive backup by which nodes are added to the controller. To counter the growth of the controllers, pruning can be uses as in DP. The iteration of the process leads to

In: m_0 = (m_1_0, ... , m_D_0)
tau = 0
m_tau = m_0
while codition_satisfied = False
	for i = 1 .. D
		m_i_tau = ExhaustiveBackup(m_i_(tau-1))
	end
	V_m_tau = ComputeValue(m_tau)
	while pruning == True
		for i = 1 .. D 
		m_i_tau = Prune(i, m_(-i)_tau, m_i_tau)
		UpdateController(m_i_tau)
		V_m_tau = ComputeValue(m_tau)
		end
	end
	tau = tau + 1
	if (gamma^(tau+1) |R_max| )/ (1-gamma) <= epsi
		condition_satisfied = True
	end
return m_star = m_tau

Due to the intractability of the resulting approach even in moderate sizes, various approximate approaches have been proposed, in order to keep the controller’s size tractable while maintaining good performances. Such methods can be Bounded Policy Iteration, Best-First Search, Expectation Maximization, Nonlinear Programming.

One typical approach is to reformulate the optimization problem resulting from the controller design into its dual, usually leading to a much more limited problem dimension. For example, in [CJJMST-2021] (Hypothesis: Single Agent uPOMDPs) the problem dimensions are reduced by formulating a dual problem and linearizing it. In this case, the problem is to design a FSC for Uncertain POMDPs (uPOMDPs), that is POMDPs with PD belonging to uncertainty sets. This uncertainty is used to capture epistemic uncertainty and it is base on uncountable sets of probability distributions, caused for instance by lack of data.

DICE [OKV-2008]

In general, solving a Dec-POMDP can be approached as a combinatorial optimization problem. In particular, DIrect Cross-Entropy (DICE) optimization can be used in the FH case [OKV-2008].

EM [KZ-2010]

As an alternative for determining the parameters of stochastic controllers, expectation maximization (EM) can be used as in [KZ-2010]. Rather than determining the controller parameters using optimization, the problem is reformulated as a likelihood maximization problem, then the framework is the so called planning as inference.

The problem can be represented as an infinite mixture of dynamic Bayesian networks which has one component for each time-step. For a time step ttt the network represents the probability that the maximum reward is received at its last stage. The intuition is that this quantity can be used as a substitute for the value of the controller.

The elements are binary reward variables for each stage used as probabilities p(r=1∣st,at)=(R(st,at)−Rmin)/(Rmax−Rmin)p(r=1|s_t, a_t) = (R(s_t, a_t) - R_{min})/(R_{max}-R_{min})p(r=1∣st​,at​)=(R(st​,at​)−Rmin​)/(Rmax​−Rmin​), which encode the chance of getting the highest possible reward at time ttt. This can be used to define p(r,Z∣t;θ)p(r, Z|t;\theta)p(r,Z∣t;θ) with

Relaxations

As it was said before, information space scales exponentially with the number of agents and with the time-horizon. One idea might be to exploit the POMDP setting while mitigating the complexity of the problem by lowering the dependence of individual plans. In particular, in [CSMO-2021](Hypothesis: Centralized Belief Update and Communication, Factored rewards), the approach is to solve independent POMDPs for each agent but still foster online cooperation during the execution phase by distributing the individual policies using auctions.

Their approach builds on two decoupled mechanisms: a decentralized data fusion filter and a POMDP auction. The former allows the agents to share information and build a joint belief like in Multi-agent POMDPs, the latter is used to assign the different behaviours to the agents in a cooperative manner, where for behaviours they mean different policies to follow given the current belief. Each single agent will then solve its own POMDP using the selected behaviour. This is done by factorizing the rewards as well with

R(s,a)=R1loc(s1,a1)+⋯+RDloc(sD,aD)+Rglob(s,a)R(s,a) = R_1^{loc}(s_1,a_1) + \dots + R_D^{loc}(s_D,a_D) + R^{glob}(s,a)R(s,a)=R1loc​(s1​,a1​)+⋯+RDloc​(sD​,aD​)+Rglob(s,a)

One other option might be to assume independence as local full observability, as in [ZBS-2020](Hypothesis: Full Observability, Continuous Space), which makes use of Gaussian Processes (GP) both for estimating other agents’ actions and for for modelling individual actions, which provides safety bounds as well.

One interesting reformulation based on information theoretic approaches is to view the decentralization as a compression problem, as in [DKT-2017] (Hypothesis: Full Observability, Offline Centralized Training). The agent’s optimal action is found by training each agent to replicate what it would have done with full information. That is, the vector of state variables is compressed and the agent must decompress it to compute an estimate of the optimal policy a^i≈ai∗\hat{a}_i \thickapprox a^*_ia^i​≈ai∗​, in particular by learning a parametrized Markov control policy a^i=π^i(si)\hat{a}_i = \hat{\pi}_i(s_i)a^i​=π^i​(si​) via regression. The policies are learned from a data set containing histories of local states and optimal actions computed by an offline centralized optimization problem.
